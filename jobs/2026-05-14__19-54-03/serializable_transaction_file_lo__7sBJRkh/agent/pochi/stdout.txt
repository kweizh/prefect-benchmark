### `flow.py` — key design points

```python
lock_manager = FileSystemLockManager(lock_files_directory=LOCK_DIR)
store = ResultStore(lock_manager=lock_manager)

with transaction(
    isolation_level=IsolationLevel.SERIALIZABLE,
    store=store,
    key="counter-file-lock",
):
    current = read_counter()
    write_counter(current + 1)
```

* **`FileSystemLockManager`** — stores advisory lock files under `/home/user/prefect-project/locks`. Every process that enters the transaction acquires an exclusive lock on the key `"counter-file-lock"` before doing any work.
* **`IsolationLevel.SERIALIZABLE`** — forces each transaction to run as if it were the only one executing; no two transactions can overlap their critical section.
* **`ResultStore`** — the bridge between `transaction()` and the lock manager (Prefect 3's `transaction()` accepts a `store` argument rather than a `lock_manager` argument directly).
* The two `@task` functions (`read_counter` / `write_counter`) perform the actual file I/O inside the protected region.

### `run_concurrent.py` — key design points

* Uses `asyncio.gather` to launch all 5 subprocesses simultaneously, so they truly race to acquire the lock.
* Each subprocess imports and calls `concurrent_file_modifier()` directly.
* After all processes finish it reads `counter.txt` and asserts the value equals 5, printing `SUCCESS` or `FAILURE` accordingly.
