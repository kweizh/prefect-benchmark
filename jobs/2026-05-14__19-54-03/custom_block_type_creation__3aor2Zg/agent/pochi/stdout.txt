Three scripts have been created under `/home/user/myproject` and verified against a running Prefect 3.7.0 instance:

### `custom_block.py`
Defines the `DatabaseConfig` block:
- `host: str`
- `port: int = 5432`
- `password: SecretStr` â€” stored as a Pydantic secret, never exposed in plaintext

```python
from prefect.blocks.core import Block
from pydantic import SecretStr

class DatabaseConfig(Block):
    host: str
    port: int = 5432
    password: SecretStr
```

---

### `setup_block.py`
Registers the block type schema with the local Prefect SQLite database and saves an instance named `my-db-config`:

```python
await DatabaseConfig.register_type_and_schema()
db_config = DatabaseConfig(host="localhost", port=5432, password="supersecret")
await db_config.save("my-db-config", overwrite=True)
```

Running `python3 setup_block.py` outputs:
```
Block 'my-db-config' saved successfully.
```
(also written to `output.log`)

---

### `load_block.py`
Loads the saved block by name and prints its `host`:

```python
db_config = await DatabaseConfig.load("my-db-config")
print(db_config.host)
```

Running `python3 load_block.py` outputs:
```
localhost
```   
